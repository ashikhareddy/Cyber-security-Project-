## Overview

Week 11 focuses on the physical prototype of the MMHN network. In the previous lab session the group successfully established basic connectivity between the TP-Link Archer C24 router, the Cisco Catalyst 3560-CX switch, and a university PC. Connectivity was verified through a successful ping test with 0% packet loss.

This week the group aims to move beyond basic connectivity and bring the physical prototype closer to the logical design implemented in Cisco Packet Tracer. The goal is to configure the physical devices with proper IP addressing, security settings, and VLAN segmentation where possible, and to record the configuration process as video evidence.

---

## Goals for This Session

**1. Assign correct IP addresses**
Configure static IP addresses on the physical devices to match the MMHN addressing scheme. This includes setting the correct subnet masks and default gateways on connected devices so they reflect the design documented in the project.

**2. Add security protocols**
Apply basic security configurations to the physical devices. This includes setting passwords on the Cisco switch, disabling unused ports, and configuring the TP-Link router with WPA2 wireless security and access restrictions where possible.

**3. Configure VLANs**
Attempt VLAN configuration on the Cisco Catalyst 3560-CX switch to segment traffic into at least two separate VLANs, mirroring the Admin and Clinical VLAN separation from the logical design. This is dependent on available time and equipment constraints.

**4. Record video of configuration and setup**
Record a short video documenting the physical setup, cable connections, and CLI configuration steps on the Cisco switch. This serves as verifiable evidence of the group's hands-on work with real networking equipment.

**5. Configure IP routing**
Set up basic IP routing between the router and switch so devices on different subnets can communicate. This validates that the physical prototype can replicate the inter-VLAN routing behaviour demonstrated in Packet Tracer.

---

## Expected Outcome

By the end of this session the group expects to have a physical prototype that demonstrates correct IP addressing, basic security hardening, VLAN segmentation, and verified inter-device communication through ping tests and CLI output screenshots.
